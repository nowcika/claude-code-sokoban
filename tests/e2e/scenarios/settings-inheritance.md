# 설정 상속 & 교육 패널 시나리오

**기능 분류**: 교육 콘텐츠 (정적 정보 영역) / L4 상속 레벨
**관련 요소**:
- "📂 방 = Claude Code 설정 구조" 패널 (`#legend` 영역, 방=설정 파일 매핑)
- "⚙️ 설정 상속 (우선순위)" 패널 (상속 우선순위 시각화)
- "？ 게임 설명 & Claude Code 배우기" 본문(도움말 모달과 공유)
**관련 레벨**: L4 · 스코프 & 상속 (방 8)

> 검증됨: 우측에 설정 구조 범례 패널과 상속 우선순위 패널이 정적으로 표시됨. 상속 패널은 `~/.claude/settings.json` → `.claude/settings.json` → `.claude/settings.local.json` → `managed-settings.json`(최우선, 못 덮음) 순서를 시각화. 메모리 상속(`~/.claude/CLAUDE.md` → `./CLAUDE.md` → 하위 폴더 `CLAUDE.md`)도 안내.

> 이 영역은 비개발자 학습용 콘텐츠다. 시나리오는 "정보가 정확히 표시되는지"와 "L4가 상속 개념을 게임으로 구현하는지"를 검증한다.

## SC-INH-01 · 설정 구조 범례 패널 표시

- **목적**: "방 = 설정 구조" 범례가 8종 설정 파일/디렉토리를 올바르게 보여주는지 확인한다.
- **전제조건**: 페이지 로드 완료.
- **테스트 단계**:
  1. 범례 패널에서 `settings.json`, `CLAUDE.md`, `.mcp.json`, `.claude/commands/`, `.claude/agents/`, `.gitignore` 등 항목과 각 설명(훅·권한·메모리·MCP·슬래시 명령·서브에이전트·무시 규칙)을 확인한다.
  2. 각 항목의 아이콘/툴팁 텍스트가 깨지지 않았는지 확인한다.
- **예상 결과**: 설정 파일 매핑 정보가 정확히 표시된다.
- **실패 조건**: 항목 누락, 설명 텍스트 깨짐.
- **우선순위**: medium
- **예상 실행시간**: 약 5초

## SC-INH-02 · 상속 우선순위 패널 순서 정확성

- **목적**: 상속 패널이 "넓은 범위 → 좁은 범위(우선)" 순서를 올바르게 시각화하는지 확인한다.
- **전제조건**: 페이지 로드 완료.
- **테스트 단계**:
  1. 상속 패널의 순서를 위에서 아래로 읽는다:
     `~/.claude/settings.json`(사용자 전역) → `.claude/settings.json`(프로젝트 공유) → `.claude/settings.local.json`(프로젝트 개인) → `managed-settings.json`(관리자 정책, 최우선).
  2. 각 단계 사이에 "▼ 덮어씀" 표시가 있는지 확인한다.
  3. `managed-settings.json`이 "최우선·못 덮음"으로 안내되는지 확인한다.
- **예상 결과**: 4단계 우선순위와 덮어쓰기 방향이 정확히 표시된다.
- **실패 조건**: 순서가 뒤바뀜, 단계 누락, 관리자 정책 우선순위 오기.
- **우선순위**: medium
- **예상 실행시간**: 약 5초

## SC-INH-03 · 메모리 상속 안내 정확성

- **목적**: 메모리(CLAUDE.md) 상속 순서 안내가 올바른지 확인한다.
- **전제조건**: 페이지 로드 완료.
- **테스트 단계**:
  1. 상속 패널 하단에서 `~/.claude/CLAUDE.md` → `./CLAUDE.md` → 하위 폴더 `CLAUDE.md` 순서 안내를 확인한다.
  2. "가까운 파일이 우선·누적" 설명이 있는지 확인한다.
- **예상 결과**: 메모리 상속 규칙이 정확히 안내된다.
- **실패 조건**: 잘못된 순서, 설명 누락.
- **우선순위**: low
- **예상 실행시간**: 약 4초

## SC-INH-04 · L4 (스코프 & 상속) 레벨 플레이 가능성

- **목적**: 상속/스코프 개념을 게임화한 L4가 정상적으로 풀 수 있는 보드인지 확인한다.
- **전제조건**: 페이지 로드 완료.
- **테스트 단계**:
  1. `#level`을 L4로 선택한다.
  2. 보드가 렌더링되고 `realTotal()` > 0인지 확인한다.
  3. 모든 목표 짐이 도달 가능한지(데드락 없이) 확인한다.
  4. L4를 끝까지 풀어 승리 모달이 뜨는지 확인한다(또는 `placedCount()===realTotal()` 도달).
- **예상 결과**: L4가 풀 수 있는 유효 보드로 제공되고 클리어 가능하다.
- **실패 조건**: 풀 수 없는 보드, 전환 시 예외.
- **우선순위**: high
- **예상 실행시간**: 약 18초

## SC-INH-05 · L4 클리어 후 다음 동작 (마지막 레벨 처리)

- **목적**: 마지막 레벨(L4) 클리어 시 "다음" 동작이 정의된 대로(완주 안내/L1 회귀 등) 처리되는지 확인한다.
- **전제조건**: L4 클리어로 승리 모달 노출.
- **테스트 단계**:
  1. L4를 클리어한다.
  2. `#winNext`를 클릭한다.
  3. 동작(완주 메시지, L1로 순환, 또는 버튼 비활성)을 관찰한다.
- **예상 결과**: 마지막 레벨 이후 동작이 명확하고 일관되다.
- **실패 조건**: 예외 발생, 존재하지 않는 L5로 진입 시도.
- **우선순위**: medium
- **예상 실행시간**: 약 8초

## SC-INH-06 · 교육 패널과 도움말 본문 일관성

- **목적**: 우측 정적 교육 패널과 도움말 모달의 설정 파일 설명이 서로 모순되지 않는지 확인한다.
- **전제조건**: 페이지 로드 완료.
- **테스트 단계**:
  1. 도움말 모달의 설정 파일 목록과 우측 범례 패널의 항목을 비교한다.
  2. 동일 파일에 대한 설명이 일치하는지 확인한다.
- **예상 결과**: 두 곳의 교육 정보가 일관된다.
- **실패 조건**: 같은 파일에 상충되는 설명.
- **우선순위**: low
- **예상 실행시간**: 약 6초

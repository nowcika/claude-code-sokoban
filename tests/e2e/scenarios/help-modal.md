# 도움말 모달 시나리오

**기능 분류**: 온보딩 / 도움말
**관련 요소**: `#helpOverlay`(클래스 `overlay`, 표시 시 `show` 추가), `#helpClose`, "시작하기" 버튼, `#helpBtn`
**관련 저장소**: localStorage `cc_sokoban_seen`

> 동작 메커니즘(검증됨): 오버레이는 기본 `class="overlay"`(display:none)이며, 열릴 때 `show` 클래스가 추가되어 `display:flex`가 된다. 첫 방문 시 자동으로 `show`가 붙어 노출된다.

---

## SC-HELP-01 · 첫 방문 시 도움말 자동 노출

- **목적**: 처음 방문한 사용자에게 도움말 모달이 자동으로 표시되는지 확인한다.
- **전제조건**: localStorage `cc_sokoban_seen` 키가 없는 상태(신규 방문 / 시크릿 창 / 저장소 초기화).
- **테스트 단계**:
  1. localStorage를 비운 상태에서 페이지를 접속한다.
  2. `#helpOverlay`가 `show` 클래스를 갖고 `display:flex`인지 확인한다.
  3. 모달 안에 "？ 게임 설명 & Claude Code 배우기" 제목과 "시작하기" 버튼이 보이는지 확인한다.
- **예상 결과**: 도움말 모달이 자동으로 화면에 표시된다.
- **실패 조건**: 모달이 표시되지 않음, "시작하기" 버튼 부재.
- **우선순위**: high
- **예상 실행시간**: 약 5초

---

## SC-HELP-02 · "시작하기" 버튼으로 닫기 + 재방문 시 미노출

- **목적**: 도움말을 닫으면 사라지고, 이후 방문에서는 자동으로 뜨지 않는지 확인한다.
- **전제조건**: SC-HELP-01 상태(모달 노출 중).
- **테스트 단계**:
  1. "시작하기" 버튼을 클릭한다.
  2. `#helpOverlay`에서 `show`가 제거되어 `display:none`인지 확인한다.
  3. localStorage `cc_sokoban_seen` 값이 "1"로 저장됐는지 확인한다.
  4. 페이지를 새로고침한다.
  5. 새로고침 후 `#helpOverlay`가 자동으로 노출되지 않는지 확인한다.
- **예상 결과**: 모달이 닫히고, `cc_sokoban_seen="1"`이 저장되며, 재방문 시 자동 노출되지 않는다.
- **실패 조건**: 클릭 후에도 모달이 남아 있음, 플래그 미저장, 재방문 시 다시 자동 노출.
- **우선순위**: high
- **예상 실행시간**: 약 8초

---

## SC-HELP-03 · 도움말 버튼(#helpBtn)으로 수동 재오픈

- **목적**: 이미 방문한 사용자가 도움말 버튼으로 모달을 다시 열 수 있는지 확인한다.
- **전제조건**: `cc_sokoban_seen="1"` (모달 닫힘 상태).
- **테스트 단계**:
  1. 상단 `#helpBtn`("？ 도움말")을 클릭한다.
  2. `#helpOverlay`가 다시 노출되는지 확인한다.
- **예상 결과**: 도움말 모달이 다시 표시된다.
- **실패 조건**: 버튼 클릭 후 모달이 열리지 않음.
- **우선순위**: medium
- **예상 실행시간**: 약 4초

---

## SC-HELP-04 · 닫기(X / #helpClose)로 닫기

- **목적**: 모달의 닫기(X) 컨트롤로도 모달을 닫을 수 있는지 확인한다.
- **전제조건**: 도움말 모달 노출 중.
- **테스트 단계**:
  1. `#helpClose`(X) 요소를 클릭한다.
  2. `#helpOverlay`가 숨겨지는지 확인한다.
- **예상 결과**: 모달이 닫힌다.
- **실패 조건**: 닫기 컨트롤이 동작하지 않음.
- **우선순위**: medium
- **예상 실행시간**: 약 4초

## SC-HELP-05 · 도움말 내용 교육 정보 정확성

- **목적**: 모달이 Claude Code 설정 파일 설명(교육 목적)을 올바르게 담는지 확인한다.
- **전제조건**: 도움말 모달 노출 중.
- **테스트 단계**:
  1. 모달 본문에서 `settings.json`, `CLAUDE.md`, `.mcp.json`, `.claude/commands/`, `.claude/agents/`, `.claude/skills/`, `settings.local.json`, `.gitignore` 항목이 모두 나열되는지 확인한다.
- **예상 결과**: 8개 설정 파일/디렉토리 설명이 모두 표시된다.
- **실패 조건**: 항목 누락 또는 설명 텍스트 깨짐.
- **우선순위**: low
- **예상 실행시간**: 약 5초

## SC-HELP-06 · (엣지) 모달 열린 상태에서 게임 조작 차단 여부

- **목적**: 도움말 모달이 떠 있는 동안 키 입력이 게임 보드로 전달되어 의도치 않게 이동하지 않는지 확인한다.
- **전제조건**: 도움말 모달 노출 중, 이동 수 0.
- **테스트 단계**:
  1. 모달이 열린 상태에서 ArrowUp/ArrowRight 등을 입력한다.
  2. `#moveCount`가 0으로 유지되는지 확인한다.
- **예상 결과**: 모달 표시 중에는 보드 이동이 발생하지 않거나(권장), 발생하더라도 명확히 정의된 동작을 따른다.
- **실패 조건**: 정의되지 않은 동작으로 모달 뒤 보드가 임의로 움직임.
- **우선순위**: low
- **예상 실행시간**: 약 5초
- **비고**: 차단 동작이 미구현일 수 있으므로, 결과를 "기대 동작 명세화"로 기록한다.

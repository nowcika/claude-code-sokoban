# 레벨 선택 시나리오

**기능 분류**: 레벨 / 난이도 구성
**관련 요소**: `#level` (combobox "레벨 선택")
**관련 전역 API**: `setLevel(i)`, `realTotal()`, `placedCount()`
**레벨 구성**:
- L1 · 설정 첫걸음 (방 6)
- L2 · 설정 심화 (방 8)
- L3 · 전체 생태계 (방 12)
- L4 · 스코프 & 상속 (방 8)

> 검증됨: `setLevel(2)` 호출 시 `#level` 셀렉트 value가 "2"로 갱신되고 보드가 재구성된다(`placedCount` 0으로 리셋). 각 레벨의 목표 짐 수는 난이도/잡동사니 설정과 무관하게 레벨 자체에 의해 결정된다.

---

## SC-LVL-01 · 레벨 셀렉트 4종 옵션 존재

- **목적**: 레벨 드롭다운에 4개 레벨이 모두 제공되는지 확인한다.
- **전제조건**: 페이지 로드 완료.
- **테스트 단계**:
  1. `#level` 옵션 목록을 읽는다.
  2. L1, L2, L3, L4 4개 옵션과 각 라벨(방 개수 포함)을 확인한다.
  3. 기본 선택값이 L1(index 0)인지 확인한다.
- **예상 결과**: 4개 옵션 존재, 기본값 L1.
- **실패 조건**: 옵션 수가 4가 아님, 기본 선택값이 L1이 아님.
- **우선순위**: high
- **예상 실행시간**: 약 4초

## SC-LVL-02 · 레벨 변경 시 보드 재구성 및 통계 리셋

- **목적**: 레벨을 바꾸면 보드가 새로 그려지고 이동 수·정리 수가 초기화되는지 확인한다.
- **전제조건**: L1에서 짐을 1개 이상 옮겨 통계가 0이 아닌 상태.
- **테스트 단계**:
  1. L1에서 `tryMove`로 몇 번 이동해 `#moveCount` > 0으로 만든다.
  2. `#level`을 L3로 변경한다(드롭다운 선택 또는 `setLevel(2)`).
  3. `#moveCount`, `#placedCount`를 확인한다.
  4. 보드 화면(스크린샷)이 이전과 달라졌는지 확인한다.
- **예상 결과**:
  - 레벨 변경 후 `#moveCount` = "0".
  - `#placedCount` = "0/N" (해당 레벨 목표 수, 정리 0).
  - 보드 레이아웃이 새 레벨로 갱신됨.
- **실패 조건**: 통계가 리셋되지 않음, 보드가 갱신되지 않음.
- **우선순위**: critical
- **예상 실행시간**: 약 8초

## SC-LVL-03 · 각 레벨의 목표 짐 수 검증

- **목적**: 레벨별 목표 짐 수(realTotal)가 정의대로인지 확인한다.
- **전제조건**: 기본 난이도(보통), 장애물/잡동사니 없음.
- **테스트 단계**:
  1. L1로 설정 후 `realTotal()`을 읽는다.
  2. L2, L3, L4 각각에 대해 `setLevel`로 전환 후 `realTotal()`을 읽는다.
  3. 각 값이 `#placedCount`의 분모와 일치하는지 확인한다.
- **예상 결과**: 레벨별 `realTotal()`이 일관되며(예: L1=7), `#placedCount` 분모와 동일하다.
- **실패 조건**: `realTotal()`과 `#placedCount` 분모 불일치, 0 또는 비정상 값.
- **우선순위**: high
- **예상 실행시간**: 약 8초
- **비고**: 측정 기준선 — L1 `realTotal()=7`. 다른 레벨 값은 실행 시 수집해 회귀 기준으로 기록.

## SC-LVL-04 · L4 (스코프 & 상속) 진입 및 특수 구조 확인

- **목적**: 상속/스코프 개념을 다루는 L4가 정상 로드되는지 확인한다.
- **전제조건**: 페이지 로드 완료.
- **테스트 단계**:
  1. `#level`을 L4로 선택한다.
  2. 보드가 정상 렌더링되고 `realTotal()`이 0보다 큰지 확인한다.
  3. 우측 "⚙️ 설정 상속 (우선순위)" 패널이 표시되는지 확인한다(상속 교육 컨텍스트).
- **예상 결과**: L4가 오류 없이 로드되고 목표 짐이 존재한다.
- **실패 조건**: L4 전환 시 예외 발생, 보드 미렌더링.
- **우선순위**: medium
- **예상 실행시간**: 약 6초

## SC-LVL-05 · (경계) 잘못된 레벨 인덱스 호출

- **목적**: `setLevel`에 범위 밖 인덱스를 줘도 게임이 깨지지 않는지 확인한다.
- **전제조건**: 페이지 로드 완료.
- **테스트 단계**:
  1. `setLevel(-1)`을 호출한다.
  2. `setLevel(99)`를 호출한다.
  3. 게임이 예외로 멈추지 않고, `#level` 값이 유효 범위(0~3)에 머무는지 확인한다.
- **예상 결과**: 잘못된 인덱스는 무시되거나 클램프되어 게임이 안정적으로 유지된다.
- **실패 조건**: 미처리 예외로 게임 중단, 캔버스 멈춤.
- **우선순위**: low
- **예상 실행시간**: 약 5초

---

## SC-LVL-06 · 레벨 변경이 진행 중 게임을 버리는지 확인

- **목적**: 풀이 도중 레벨을 바꿀 때 현재 진행이 적절히 폐기되는지 확인한다(데이터 손실 안내 불필요 여부).
- **전제조건**: L1에서 짐 일부를 목표에 올린 상태(`placedCount() > 0`).
- **테스트 단계**:
  1. L1에서 짐 1개 이상을 목표에 놓는다.
  2. L2로 전환한다.
  3. 다시 L1로 돌아온다.
  4. L1의 진행 상태가 초기화되어 있는지(저장되지 않음) 확인한다.
- **예상 결과**: 레벨 전환 시 진행이 저장되지 않고 새로 시작된다(설계상 정상).
- **실패 조건**: 전환 후 상태 꼬임(이전 짐 위치 잔존, 통계 불일치).
- **우선순위**: low
- **예상 실행시간**: 약 8초

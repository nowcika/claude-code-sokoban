# 랭킹 & 영속성(localStorage) 시나리오

**기능 분류**: 점수 기록 / 영속성
**관련 요소**: `#rankBtn`(🏆 랭킹), 랭킹 패널 `#rankBody`/랭킹 헤딩(예: "🏆 랭킹 · L1 / 보통"), 승리 모달 내 `#playerName`(이름), `#saveScore`(저장)
**관련 저장소**: localStorage 키 `sokoban_cc_scores_v1`
**분리 기준**: 레벨 × 난이도 × 장애물 × 잡동사니별로 별도 랭킹 저장

> 검증됨: 신규 방문 시 `sokoban_cc_scores_v1` = null(기록 없음). 랭킹 패널은 "아직 기록이 없어요" 빈 상태를 표시. 랭킹 헤딩에 현재 레벨/난이도가 표기됨.

---

## SC-RANK-01 · 초기 빈 랭킹 표시

- **목적**: 기록이 없을 때 랭킹 패널이 빈 상태 메시지를 보여주는지 확인한다.
- **전제조건**: localStorage `sokoban_cc_scores_v1` 없음(초기화 상태).
- **테스트 단계**:
  1. 저장소를 비운 채 접속한다.
  2. 랭킹 패널(`#rankBody`)에 "아직 기록이 없어요"가 표시되는지 확인한다.
  3. 랭킹 헤딩이 현재 구성(예: "L1 / 보통")을 반영하는지 확인한다.
- **예상 결과**: 빈 상태 메시지와 현재 구성 라벨이 표시된다.
- **실패 조건**: 빈 상태 메시지 부재, 잘못된 구성 라벨.
- **우선순위**: high
- **예상 실행시간**: 약 5초

---

## SC-RANK-02 · 클리어 후 점수 저장 (이름 + #saveScore)

- **목적**: 클리어 후 이름을 입력하고 저장하면 랭킹에 기록되는지 확인한다.
- **전제조건**: 한 퍼즐을 클리어해 승리 모달 노출 중.
- **테스트 단계**:
  1. 승리 모달에서 `#playerName`에 이름(예: "테스터")을 입력한다.
  2. `#saveScore`를 클릭한다.
  3. 랭킹 패널에 해당 이름·이동 수가 추가됐는지 확인한다.
  4. localStorage `sokoban_cc_scores_v1`에 항목이 저장됐는지 확인한다.
- **예상 결과**: 이름·이동 수가 랭킹과 localStorage에 저장된다.
- **실패 조건**: 저장 무반응, 랭킹/저장소 미반영.
- **우선순위**: critical
- **예상 실행시간**: 약 12초

---

## SC-RANK-03 · 이동 수 기준 정렬 (적은 순)

- **목적**: 랭킹이 이동 수 오름차순(적은 이동이 상위)으로 정렬되는지 확인한다.
- **전제조건**: 같은 구성에서 서로 다른 이동 수로 2개 이상 기록 저장.
- **테스트 단계**:
  1. 동일 레벨/난이도/장애물/잡동사니 구성에서 이동 수가 다른 두 번의 클리어를 저장한다.
  2. 랭킹 표의 순서가 이동 수 적은 순인지 확인한다.
- **예상 결과**: 이동 수가 적은 기록이 상위에 표시된다.
- **실패 조건**: 정렬이 뒤바뀜, 정렬 누락.
- **우선순위**: high
- **예상 실행시간**: 약 15초

---

## SC-RANK-04 · 구성별 랭킹 분리 (레벨×난이도×장애물×잡동사니)

- **목적**: 서로 다른 구성의 기록이 섞이지 않고 각 구성별로 분리 저장·표시되는지 확인한다.
- **전제조건**: 두 가지 다른 구성에서 각각 클리어/저장.
- **테스트 단계**:
  1. L1/보통/장애물 없음/잡동사니 없음에서 클리어 후 저장한다.
  2. 난이도를 "어려움"으로 바꾸고 다시 클리어 후 저장한다.
  3. 랭킹 패널을 보통/어려움 각 구성으로 전환하며 각 기록이 분리돼 표시되는지 확인한다.
  4. 첫 구성 랭킹에 두 번째 기록이 섞이지 않는지 확인한다.
- **예상 결과**: 구성마다 독립적인 랭킹 목록을 갖는다.
- **실패 조건**: 다른 구성 기록이 한 목록에 섞임.
- **우선순위**: high
- **예상 실행시간**: 약 20초

---

## SC-RANK-05 · 새로고침 후 기록 영속

- **목적**: 저장된 랭킹이 페이지 새로고침 후에도 유지되는지 확인한다.
- **전제조건**: SC-RANK-02로 기록 1개 이상 저장.
- **테스트 단계**:
  1. 점수를 저장한다.
  2. 페이지를 새로고침한다.
  3. 같은 구성의 랭킹 패널에 기록이 그대로 보이는지 확인한다.
- **예상 결과**: 새로고침 후에도 랭킹이 유지된다(localStorage 영속).
- **실패 조건**: 새로고침 후 기록 소실.
- **우선순위**: critical
- **예상 실행시간**: 약 8초

## SC-RANK-06 · 랭킹 버튼(#rankBtn) 동작

- **목적**: 🏆 랭킹 버튼이 랭킹 보기를 열거나 해당 패널로 포커스/스크롤하는지 확인한다.
- **전제조건**: 페이지 로드 완료.
- **테스트 단계**:
  1. `#rankBtn`을 클릭한다.
  2. 랭킹 패널이 보이거나 강조/스크롤되는지 확인한다.
- **예상 결과**: 랭킹 영역이 사용자에게 노출/강조된다.
- **실패 조건**: 버튼 무반응.
- **우선순위**: medium
- **예상 실행시간**: 약 4초
- **비고**: 별도 `#rankOverlay`는 없음 — 랭킹은 인라인 패널(`#rankBody`)로 확인됨. 버튼은 패널 표시/스크롤 용도로 추정.

## SC-RANK-07 · (엣지) 빈 이름으로 저장 시도

- **목적**: 이름 미입력 시 저장 처리(거부 또는 기본값)가 일관적인지 확인한다.
- **전제조건**: 클리어 후 승리 모달, `#playerName` 비움.
- **테스트 단계**:
  1. 이름을 비운 채 `#saveScore`를 클릭한다.
  2. 저장이 거부되는지, 또는 기본 이름(예: "익명")으로 저장되는지 확인한다.
- **예상 결과**: 정의된 동작(거부 또는 기본값)이 일관되게 적용된다.
- **실패 조건**: 예외 발생, 깨진 빈 항목 저장.
- **우선순위**: low
- **예상 실행시간**: 약 6초

---

## SC-RANK-08 · (엣지) localStorage 손상/차단 내성

- **목적**: localStorage가 비활성(시크릿/차단)이거나 손상돼도 게임이 동작하는지 확인한다.
- **전제조건**: localStorage 접근 차단 또는 `sokoban_cc_scores_v1`에 잘못된 JSON 주입.
- **테스트 단계**:
  1. `sokoban_cc_scores_v1`에 깨진 문자열(`"{invalid"`)을 넣는다.
  2. 페이지를 새로고침한다.
  3. 게임이 예외로 멈추지 않고 빈 랭킹으로 폴백하는지 확인한다.
- **예상 결과**: 손상 데이터는 무시되고 게임이 정상 동작한다.
- **실패 조건**: 파싱 오류로 게임 초기화 실패.
- **우선순위**: low
- **예상 실행시간**: 약 7초

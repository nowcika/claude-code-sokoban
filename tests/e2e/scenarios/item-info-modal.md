# 짐 정보(교육) 모달 시나리오

**기능 분류**: 교육 콘텐츠 / 정보 모달
**관련 요소**: `#infoOverlay`(클래스 `overlay`, 표시 시 `show`), `#infoClose`
**관련 전역 API**: `openItemInfo(box)`
**관련 입력**: 보드 위 짐을 마우스로 클릭

> 검증됨: `#infoOverlay`는 기본 `class="overlay"`(display:none)이며 열릴 때 `show`가 추가됨. `openItemInfo`는 전역 함수로 노출됨. 짐 클릭 시 해당 짐의 정체(어느 설정 파일/방에 놓아야 하는지, 실제 Claude Code 사용법)를 설명하는 모달이 뜸.

---

## SC-INFO-01 · 짐 클릭 시 정보 모달 노출

- **목적**: 보드의 짐을 클릭하면 교육 설명 모달이 표시되는지 확인한다.
- **전제조건**: L1, 모달 닫힘 상태. 짐의 화면 좌표 확보 필요(캔버스 클릭).
- **테스트 단계**:
  1. 보드 위 한 짐의 셀 중심 화면 좌표를 계산한다.
  2. 해당 좌표를 클릭한다(또는 검증용으로 `openItemInfo(box)` 호출).
  3. `#infoOverlay`에 `show`가 추가되어 `display:flex`인지 확인한다.
  4. 모달 안에 해당 짐의 설명 텍스트(설정 파일/방, 사용법)가 있는지 확인한다.
- **예상 결과**: 짐 클릭 시 정보 모달이 열리고 교육 내용이 표시된다.
- **실패 조건**: 클릭 무반응, 모달 미노출, 내용 비어 있음.
- **우선순위**: high
- **예상 실행시간**: 약 8초

---

## SC-INFO-02 · 정보 모달 닫기 (#infoClose)

- **목적**: 정보 모달을 닫기 버튼으로 닫을 수 있는지 확인한다.
- **전제조건**: SC-INFO-01로 정보 모달이 열린 상태.
- **테스트 단계**:
  1. `#infoClose`를 클릭한다.
  2. `#infoOverlay`에서 `show`가 제거되어 `display:none`인지 확인한다.
- **예상 결과**: 모달이 닫힌다.
- **실패 조건**: 닫기 무반응, 모달 잔존.
- **우선순위**: medium
- **예상 실행시간**: 약 4초

---

## SC-INFO-03 · 짐 종류별 설명 내용 일치

- **목적**: 서로 다른 짐을 클릭하면 각기 다른(짐 정체에 맞는) 설명이 나오는지 확인한다.
- **전제조건**: L1 (여러 종류의 짐 존재: 훅 🪝, 권한 🔐, MCP 🔌 등).
- **테스트 단계**:
  1. 짐 A를 클릭해 설명 텍스트를 읽고 모달을 닫는다.
  2. 다른 종류의 짐 B를 클릭해 설명 텍스트를 읽는다.
  3. 두 설명이 서로 다르며 각 짐의 목표 설정 파일을 올바르게 안내하는지 확인한다.
- **예상 결과**: 짐마다 고유한 교육 설명이 표시된다(예: 훅 짐 → `settings.json` 안내).
- **실패 조건**: 모든 짐이 동일 설명, 잘못된 파일 안내.
- **우선순위**: medium
- **예상 실행시간**: 약 10초

## SC-INFO-04 · 정보 모달 표시 중 게임 진행 영향 없음

- **목적**: 정보 모달이 단순 정보 표시이며 짐 위치/이동 수를 바꾸지 않는지 확인한다.
- **전제조건**: 이동 수와 정리 수가 특정 값인 상태.
- **테스트 단계**:
  1. 현재 `#moveCount`, `#placedCount`를 기록한다.
  2. 짐을 클릭해 모달을 열고 닫는다.
  3. 두 통계값이 변하지 않았는지 확인한다.
- **예상 결과**: 정보 모달 조작은 게임 상태에 영향이 없다.
- **실패 조건**: 모달 열고 닫은 뒤 통계가 변함.
- **우선순위**: medium
- **예상 실행시간**: 약 6초

## SC-INFO-05 · (엣지) 잡동사니(📦) 클릭 시 동작

- **목적**: 가짜 짐(잡동사니)을 클릭했을 때의 동작(설명 노출 여부)을 명세·검증한다.
- **전제조건**: `#decoys` = "조금/많이"인 보드.
- **테스트 단계**:
  1. 잡동사니 짐을 클릭한다.
  2. 모달이 뜨는지, 또는 "이건 치워야 하는 가짜 짐" 류 안내가 나오는지 확인한다.
- **예상 결과**: 정의된 동작(설명 모달 또는 무동작)이 일관되게 발생한다.
- **실패 조건**: 예외 발생, 진짜 짐 설명이 잘못 노출.
- **우선순위**: low
- **예상 실행시간**: 약 6초
- **비고**: 잡동사니 클릭 동작은 구현 의존 — 관찰된 동작을 기준선으로 기록.

---

## SC-INFO-06 · (엣지) 빈 칸/벽 클릭 시 무반응

- **목적**: 짐이 없는 칸이나 벽을 클릭해도 정보 모달이 뜨지 않는지 확인한다.
- **전제조건**: L1, 모달 닫힘.
- **테스트 단계**:
  1. 보드의 빈 바닥 칸 좌표를 클릭한다.
  2. `#infoOverlay`가 닫힌 상태로 유지되는지 확인한다.
- **예상 결과**: 짐이 없는 곳 클릭은 모달을 열지 않는다.
- **실패 조건**: 빈 칸 클릭으로 모달이 뜸, 예외 발생.
- **우선순위**: low
- **예상 실행시간**: 약 5초

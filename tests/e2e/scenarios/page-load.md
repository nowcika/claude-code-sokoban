# 페이지 로드 & 초기 렌더링 시나리오

**대상**: https://nowcika.github.io/claude-code-sokoban/ (동일본: http://localhost:8765/index.html)
**기능 분류**: 페이지 로드 / 초기화
**관련 요소**: `<canvas id="game">`, `#level` `#difficulty` `#obstacles` `#decoys`, 버튼 `#undoBtn #resetBtn #newBtn #rankBtn #helpBtn`, 통계 `#moveCount #placedCount`, 패널 `#legend #rankBody`
**관련 전역 API**: `realTotal()`, `placedCount()`, `setLevel()`

> 참고: 보드 내부(플레이어/짐/방)는 `<canvas>`로 그려지므로 DOM 셀렉터가 없습니다. 내부 상태는 화면 캡처(스크린샷)와 전역 함수 호출 결과(`realTotal()`, `placedCount()`)로 검증합니다.

---

## SC-LOAD-01 · 페이지 정상 로드 및 핵심 UI 노출

- **목적**: 사이트 접속 시 게임 화면과 모든 핵심 컨트롤이 정상적으로 표시되는지 확인한다.
- **전제조건**: 인터넷 연결, 지원 브라우저(Chromium 계열). localStorage 상태 무관.
- **테스트 단계**:
  1. 대상 URL로 접속한다.
  2. 페이지 제목이 "Claude Code Sokoban — 설정 정리 퍼즐"인지 확인한다.
  3. 상단 헤딩 "◣ Claude Code Sokoban"과 안내 문구가 보이는지 확인한다.
  4. `<canvas id="game">`가 화면에 존재하고 크기가 0보다 큰지 확인한다.
  5. 컨트롤(`#level`, `#difficulty`, `#obstacles`, `#decoys`, 5개 버튼)이 모두 렌더링됐는지 확인한다.
  6. 통계 영역 `#moveCount`, `#placedCount`가 보이는지 확인한다.
- **예상 결과**:
  - 제목/헤딩이 정확히 표시된다.
  - 캔버스 가로·세로가 0보다 크다(예: 약 714×413px).
  - 모든 컨트롤과 통계 텍스트가 존재한다.
- **실패 조건**: 캔버스 미표시, 제목 불일치, 컨트롤 중 하나라도 누락, 페이지 로드 실패(404/타임아웃).
- **우선순위**: critical
- **예상 실행시간**: 약 5초

## SC-LOAD-02 · 초기 통계값 검증 (이동 수 0, 정리 0/7)

- **목적**: 첫 로드 시 게임이 초기 상태(이동 0, 정리 0)로 시작하는지 확인한다.
- **전제조건**: 페이지가 로드되고 L1·보통 기본 설정.
- **테스트 단계**:
  1. 페이지 접속(도움말 모달이 떠 있으면 닫는다 — 별도 시나리오 참조).
  2. `#moveCount` 텍스트를 읽는다.
  3. `#placedCount` 텍스트를 읽는다.
  4. 전역 `realTotal()`, `placedCount()` 값을 읽는다.
- **예상 결과**:
  - `#moveCount` = "0".
  - `#placedCount` = "0/7" (L1 기본 목표 짐 7개).
  - `realTotal()` = 7, `placedCount()` = 0.
- **실패 조건**: 이동 수가 0이 아님, 정리 표시가 "0/7"이 아님, `realTotal()`이 7이 아님.
- **우선순위**: high
- **예상 실행시간**: 약 5초

## SC-LOAD-03 · 전역 JS API 노출 확인

- **목적**: 자동화·테스트에 필요한 전역 함수가 모두 노출돼 있는지 확인한다.
- **전제조건**: 페이지 로드 완료.
- **테스트 단계**:
  1. 다음 전역의 타입을 확인한다: `tryMove`, `undo`, `reset`, `newGame`, `setLevel`, `placedCount`, `realTotal`, `boxOnRightTarget`, `openItemInfo`.
  2. 각각 `typeof === 'function'`인지 확인한다.
- **예상 결과**: 위 9개 전역이 모두 함수로 존재한다.
- **실패 조건**: 함수 중 하나라도 `undefined`이거나 함수가 아님.
- **우선순위**: medium (자동화 인프라 검증용)
- **예상 실행시간**: 약 3초
- **비고**: `boxes`, `targets`, `LEVELS`, `ROOMS`, `ITEMS`, `player`, `moves`, `levelIndex`는 실제 빌드에서 모듈 스코프로 캡슐화되어 `window`에 노출되지 않을 수 있다. 상태 검증은 함수 반환값과 DOM 통계로 대체한다.

## SC-LOAD-04 · 콘솔 오류 모니터링

- **목적**: 로드 시 치명적 자바스크립트 오류가 없는지 확인한다.
- **전제조건**: 콘솔 로그 캡처 가능.
- **테스트 단계**:
  1. 콘솔 메시지를 수집하며 페이지를 로드한다.
  2. error 레벨 메시지 목록을 확인한다.
- **예상 결과**: 게임 동작을 막는 치명적 오류가 없다(외부 폰트/파비콘 등 비치명적 네트워크 오류는 허용 가능, 다만 회귀 추적을 위해 기록).
- **실패 조건**: 게임 초기화를 중단시키는 미처리 예외(캔버스 미렌더링 동반).
- **우선순위**: medium
- **예상 실행시간**: 약 5초
- **비고**: 실제 측정 시 로드 직후 콘솔 error 1건이 관찰됨 — 회귀 기준선으로 기록하고, 게임 기능이 정상이면 통과로 간주.

## SC-LOAD-05 · 반응형 레이아웃 (리사이즈)

- **목적**: 뷰포트 크기 변경 시 캔버스·컨트롤이 깨지지 않는지 확인한다.
- **전제조건**: 페이지 로드 완료.
- **테스트 단계**:
  1. 데스크톱 폭(예: 1280px)에서 레이아웃을 확인한다.
  2. 모바일 폭(예: 390px)으로 리사이즈한다.
  3. 캔버스와 컨트롤이 화면 밖으로 잘리거나 겹치지 않는지 확인한다.
- **예상 결과**: 두 크기 모두에서 캔버스가 보이고 컨트롤이 접근 가능하다.
- **실패 조건**: 컨트롤이 가려져 클릭 불가, 캔버스가 0 크기로 붕괴.
- **우선순위**: low
- **예상 실행시간**: 약 8초
